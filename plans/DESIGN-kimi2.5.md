## 1. 产品概述

### 1.1 产品定位

**nanobot** 是一款超轻量级个人 AI 助手框架，专为追求简洁、高效的用户设计。与同类产品相比，nanobot 以极简代码实现核心功能，让用户真正拥有可控、可定制的 AI 助手。

### 1.2 核心价值主张

| 维度 | 传统 AI 助手 | nanobot |
|------|-------------|---------|
| 代码量 | 43万+ 行 | ~4,000 行 |
| 可控性 | 黑盒系统 | 完全开源透明 |
| 定制成本 | 高 | 极低 |
| 部署难度 | 复杂 | 一键启动 |
| 资源占用 | 高 | 极低 |

**一句话描述**: nanobot 是「代码最少、功能最全、最懂开发者」的个人 AI 助手。

### 1.3 目标用户

- **个人开发者**: 需要代码助手、日常任务管理
- **研究人员**: 需要可修改、可实验的 AI 框架
- **技术爱好者**: 追求轻量级、自托管的 AI 解决方案
- **小型团队**: 需要低成本自动化工具

## 3. 用户场景

### 3.1 场景一: 个人编程助手

**用户**: 独立开发者小明

**场景描述**:
1. 小明在 Telegram 上向 nanobot 发送需求: "帮我创建一个 Python 爬虫，抓取豆瓣电影 Top250"
2. nanobot 生成完整代码并保存到项目目录
3. 小明要求运行测试，nanobot 执行命令并返回结果
4. 发现报错后，nanobot 自动修复代码

**价值**: 从想法到可用代码，全程对话完成，无需切换工具。

### 3.2 场景二: 智能日程管理

**用户**: 产品经理小红

**场景描述**:
1. 小红说: "每天早上 9 点提醒我看今日待办"
2. nanobot 创建周期任务，每天准时推送
3. 小红说: "周五下午 3 点提醒我准备周会材料"
4. nanobot 设置一次性提醒
5. 每 30 分钟心跳检查，自动同步日历事件

**价值**: 自然语言设置提醒，多维度任务管理。

### 3.3 场景三: 研究信息助手

**用户**: 研究生小李

**场景描述**:
1. 小李: "搜索最新的 LLM 论文"
2. nanobot 搜索并返回相关论文链接
3. 小李: "总结这篇论文的主要内容 https://..."
4. nanobot 提取网页内容并生成摘要
5. 小李: "把这篇论文的关键点记到我的笔记里"
6. nanobot 更新 MEMORY.md

**价值**: 搜索-阅读-记录，一站式研究助手。

## 8. 使用门槛

### 8.1 技术要求

| 项目 | 要求 |
|------|------|
| Python | ≥ 3.11 |
| 内存 | 最低 512MB |
| 存储 | 最低 100MB |
| 网络 | 访问 LLM API |

### 8.2 上手路径

```
第 1 步: pip install nanobot-ai      (1 分钟)
第 2 步: nanobot onboard              (1 分钟)
第 3 步: 配置 API Key                 (2 分钟)
第 4 步: nanobot agent -m "Hello"     (30 秒)
─────────────────────────────────────────────
总计: 约 5 分钟拥有自己的 AI 助手
```

## 9. 安全与隐私

### 9.1 安全特性

- 危险命令拦截 (rm -rf, format 等)
- 工作区隔离 (可选)
- 命令执行超时保护
- 敏感信息过滤

### 9.2 隐私保护

- 支持本地模型 (vLLM)
- 不收集用户数据
- 记忆文件本地存储
- 完全自托管
